description: Build a detailed plan for a change to the project
---

Please develop a plan for a change to this project using the following outlined
workflow. While executing this workflow, do **NOT** perform any actions that
will result in changes to files. 

## Requirements Gathering

Begin by asking me to describe the desired change. It's important to understand
the details of the change being requested. You should ask some initial
clarifying questions to better understand the change. Your questions should
focus on the domain-level requirements and should not ask any questions of a
technical nature.

## Technical Design

Once you have an understand of the domain-level requirements, you should begin
performing the technical design required to implement the requested change.
Begin this phase by informing the user that you have begun the technical design.
If you have questions of a technical nature, attempt to answer them yourself by
exploring the project. If you do not receive a clear answer through project
exploration, ask the user to clarify. You should only prompt the user when
necessary.

## Plan Summarization

Once you understand the domain-level requirements and the technical changes
required, you should produce the plan. The plan should describe the change in
discrete actionable steps. Reference relative files where it makes sense. The
plan should be saved to a `plans/` directory at the top-level of the project.
The name of the file should be named `foo-bar-baz.md` where `foo-bar-baz` should
be replaced with a 3-5 word description of the change.
